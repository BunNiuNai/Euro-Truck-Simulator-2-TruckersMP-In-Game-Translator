# ETS2 Chat Translator (ETS2 聊天翻译器)

项目完全开源，可放心免费使用。

Euro Truck Simulator 2 / TruckersMP 联机聊天实时翻译工具。自动将多国语言聊天消息翻译为简体中文，以半透明悬浮窗口显示在游戏中。支持中文输入 → 翻译为英文 → 一键发送到游戏聊天。

可以访问这个链接直接下载exe文件。https://github.com/BunNiuNai/ets2-translator/releases

## 功能特性

- **实时聊天翻译**：监控 TruckersMP 聊天日志，批量翻译各语言消息到简体中文
- **反向翻译发送**：在翻译器中输入中文，自动翻译为英文并通过模拟键盘发送到游戏聊天
- **自己识别**：自动检测自己的游戏 ID，跳过自己发送的消息避免重复翻译
- **批量翻译**：0.8 秒窗口内收集多条消息合并为一次 API 请求，减少 API 调用次数
- **LRU 缓存**：200 条翻译缓存，相同消息直接命中跳过 API 调用
- **多种 LLM 支持**：兼容 OpenAI API 格式（OpenAI、DeepSeek、Ollama、Groq 等）
- **连通性测试**：设置中一键测试 API 连接，启动时自动检查连通状态
- **双窗口模式**：标准窗口 / 无边框悬浮窗口，悬浮模式支持鼠标穿透和边缘拖拽缩放
- **API 用量统计**：窗口底部实时显示翻译次数、缓存命中、跳过次数和节省百分比
- **窗口位置记忆**：自动保存和恢复窗口位置与大小
- **全局热键**：可自定义组合键（如 Shift+Y）从游戏内呼出翻译器输入栏
- **系统托盘**：纯 Win32 托盘图标，右键菜单快捷操作（显示/隐藏、切换模式、鼠标穿透、设置、退出）
- **聊天俚语翻译**：内置网游缩写/俚语映射（lol→哈哈，brb→马上回来 等）

## 系统要求

- Windows 10/11
- Python 3.10+

## 快速开始

### 方式一：Python 源码运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 运行
python main.py
```

### 方式二：打包为 exe

```bash
python build_exe.py
# 输出文件: dist/ETS2_Chat_Translator.exe
```

双击运行即可，无需安装 Python 环境。

## 配置说明

首次运行会自动弹出设置窗口，也可通过右键菜单 → "Settings / 设置" 打开。

填写 API 地址、密钥和模型后，点击 **"Test / 测试连接"** 验证配置是否可用，测试通过后点击 **"Save / 保存"**。配置会自动保存，下次启动无需重新填写。

每次启动时会自动测试 API 连通性，结果显示在翻译窗口中。同时显示文档目录路径和聊天日志状态，方便排查问题。

| 配置项 | 说明 | 示例 |
| --- | --- | --- |
| API Endpoint | LLM API 地址（OpenAI 兼容格式） | `https://api.openai.com/v1/chat/completions` |
| API Key | API 密钥 | `sk-xxxxxxxx` |
| Model | 模型名称 | `gpt-4o-mini` |
| 窗口透明度 | 0.1（几乎透明）~ 1.0（完全不透明） | `0.80` |
| 字体大小 | 翻译窗口字体大小 | `12` |
| 最大消息数 | 窗口内最多显示的消息条数 | `50` |
| 你的游戏ID | 在游戏中的玩家名称（可选，可自动检测） | `PlayerName` |
| 窗口模式 | 标准窗口 / 悬浮窗口 | `悬浮窗口` |
| 鼠标穿透 | 悬浮模式下鼠标点击穿透到游戏 | 否 |
| 游戏聊天键 | 游戏中打开聊天窗口的按键 | `y` |
| 呼出输入框热键 | 全局热键，从游戏中呼出翻译器输入栏 | 点击标签后按下组合键即可 |

热键设置方式：点击设置中的热键标签，标签变为"按下组合键..."，然后直接按下键盘组合键（如 Shift+Z），自动识别并保存。

**注意：绑定按键要锁定大写！**

配置文件保存在：`文档\ETS2 Translator\config.json`（路径通过注册表获取，与 TruckersMP 一致）

## 使用方式

1. 启动 ETS2 并进入 TruckersMP 联机模式
2. 运行翻译器，窗口会以悬浮或标准模式显示
3. **查看翻译**：其他玩家的消息会自动翻译并显示在窗口中
4. **发送消息**：按全局热键（默认 Shift+Y，可自定义）呼出输入栏 → 输入中文 → 回车 → 自动翻译为英文 → 倒计时"大模型翻译完成等待 (3/2/1)" → 模拟键盘发送到游戏
5. 系统托盘图标右键可进行快捷操作：显示/隐藏、切换模式、鼠标穿透、设置、退出
6. 右键点击翻译窗口可打开设置菜单

### 核心模块详解

#### 1. 聊天日志监控 (`monitor.py`)

TruckersMP 将聊天记录写入 `文档\ETS2MP\logs\chat_<日期>_log.txt`，格式为：

```
[Global] [05:03:56] Connecting to Simulation 1 server...
[Global] [05:05:25] BoJo2k26 (1713): back :D
[Global] [21:18:55] TMP-Yunee. (A 158): hello
```

每行由 **通道标签** + **时间戳** + **玩家名(服务器字母 TMP ID)** + **消息内容** 组成。

监控策略采用**增量轮询**（类似 `tail -f`），正则解析 `\([A-Z]?\s*\d+\)` 匹配 TMP ID（适配新旧格式）。

#### 2. 批量翻译引擎 (`translator.py`)

- **批量翻译**：在 0.8 秒窗口内收集消息，用 `\n---\n` 分隔合并为一次 API 请求，减少网络开销
- **LRU 缓存**：200 条 OrderedDict 缓存，相同原文直接返回缓存结果
- **智能跳过**：统计 CJK 与拉丁字符比例，跳过已是中文的消息
- **反向翻译**：`translate_to_english()` 将用户输入的中文翻译为英文游戏聊天用语
- **错误分类**：对网络错误、认证失败 (401)、限流 (429)、服务端错误等给出中文提示

#### 3. 翻译显示窗口 (`overlay.py`)

基于 Tkinter 构建，支持两种窗口模式：

| 模式 | 特点 |
| --- | --- |
| **Standalone** | 标准窗口，有标题栏，可自由缩放 |
| **Overlay** | 无边框悬浮窗，始终置顶，支持鼠标穿透和边缘拖拽缩放 |

Overlay 模式通过 Win32 API (`SetWindowLongPtrW`) 设置窗口样式：
- `WS_EX_TOOLWINDOW`: 隐藏任务栏图标，只留系统托盘
- `WS_EX_TRANSPARENT`: 鼠标穿透（点击直达游戏）

**发送功能**：底部输入栏支持热键呼出后直接输入（`AttachThreadInput` + `SetForegroundWindow` 抢焦点），回车 → LLM 翻译为英文 → 主线程设置剪贴板 → 倒计时"大模型翻译完成等待 (3/2/1)" → 模拟键盘：聊天键 → 0.5s → 聊天键 → 0.5s → Ctrl+V → 0.2s → Enter 发送。

**全局热键**：通过 `GetAsyncKeyState` 轮询实现（50ms 间隔），检测修饰键+主键组合后呼出窗口并自动移动鼠标点击输入框，确保焦点在翻译器而非游戏。

#### 4. 键盘模拟 (`input_sender.py`)

使用 Win32 `SendInput` API 进行可靠的键盘注入：
1. 按下游戏聊天热键（如 Y）
2. 等待 0.5 秒
3. 再次按下游戏聊天热键（确保聊天框获得焦点）
4. 等待 0.5 秒
5. 发送 Ctrl+V 粘贴译文
6. 等待 0.2 秒
7. 发送 Enter 键发送消息

剪贴板操作在**主线程**通过 Tkinter 的 `clipboard_clear/append` 完成，发送前后自动保存和恢复原有剪贴板内容。

#### 5. 系统托盘 (`tray_icon.py`)

纯 ctypes + Win32 API 实现，不依赖任何第三方库：
- `Shell_NotifyIconW` 创建/管理托盘图标
- `CreateWindowExW` + 消息泵处理鼠标事件
- `CreatePopupMenu` + `TrackPopupMenu` 显示右键菜单
- GDI 程序化绘制蓝色 "T" 图标

#### 6. 主控与配置 (`main.py` + `config.py`)

- **App 类**：串联所有模块，初始化监控/翻译线程、创建窗口、设置系统托盘
- **SettingsDialog**：设置界面，支持连通性测试和热键捕获（点击标签直接按组合键）
- **config.py**：配置以 JSON 存储在 `文档\ETS2 Translator\config.json`，Documents 路径通过注册表获取

### 消息生命周期

```
1. 玩家在游戏内发送聊天
       │
2. TruckersMP 写入 chat_YYYY_MM_DD_log.txt
       │
3. ChatMonitor 检测到文件变化，增量读取新行
       │
4. 正则解析 → ChatMessage，判断是否为自己发送
       │
5. 放入 raw_queue
       │
6. Translator 取出，检查 LRU 缓存
       │   命中缓存 → 直接返回
       │   未命中 → 加入批量队列
       │
7. 批量窗口到期 → 合并发送一次 API 请求
       │
8. 翻译结果拆分后分别放入 display_queue
       │
9. OverlayWindow 每 250ms 轮询队列（空闲合并刷新）
       │
10. 增量更新 Text 组件，更新统计栏，自动滚到底部
```

### 发送消息流程

```
1. 用户在游戏中按全局热键 (如 Shift+Y) 呼出输入栏
   │   翻译器窗口置顶，AttachThreadInput 抢焦点
   │   用户可直接输入中文
       │
2. 输入中文消息，按回车
   │   输入框禁用，显示"翻译中..."
   │   后台调用 LLM 翻译中文 → 英文
       │
3. 翻译完成 → 主线程设置剪贴板(译文)
   │   倒计时显示"大模型翻译完成等待 (3/2/1)"
   │   用户在此期间切换到游戏窗口
   │   按 ESC 或点击 [ESC 取消发送] 可取消
       │
4. 倒计时结束 → input_sender 执行发送序列:
   │   模拟按下游戏聊天键 (如 Y)
   │   等待 0.5 秒
   │   再次按下游戏聊天键 (确保聊天框焦点)
   │   等待 0.5 秒
   │   Ctrl+V 粘贴译文
   │   等待 0.2 秒
   │   Enter 发送
       │
5. 发送完成后恢复原有剪贴板
   │   已发送消息显示在翻译窗口（(Sent) 标签）
```

## 常见问题

**Q: 翻译结果显示 "[网络错误] 无法连接到 API 服务器"？**
A: 请检查 API Endpoint 地址是否正确，网络是否能访问该地址。

**Q: 翻译结果显示 "[认证失败] API 密钥无效"？**
A: 请检查 API Key 是否正确填写，密钥是否过期。

**Q: 连通性测试显示 "API 地址不存在 (404)"？**
A: 请检查 API Endpoint 是否包含完整路径（如 `https://api.deepseek.com/v1/chat/completions`），以及模型名称是否正确。

**Q: 启动后显示"目录不存在"或"目录存在但无聊天日志文件"？**
A: 请确认已安装 TruckersMP 并至少进入过一次联机服务器。日志目录位于 `文档\ETS2MP\logs\`，首次进入游戏后会自动创建。

**Q: 能看到日志文件但无法识别聊天内容？**
A: 如果 TruckersMP 版本较旧，日志格式可能不同。请更新 TruckersMP 客户端到最新版本。

**Q: 为什么看不到翻译窗口？**
A: 检查窗口是否在屏幕外；右键系统托盘图标选择 "Show/Hide" 切换显示；尝试切换到标准窗口模式。

**Q: 悬浮模式下无法点击/拖拽窗口？**
A: 如果开启了鼠标穿透，窗口不接受鼠标点击。请通过系统托盘菜单关闭点击穿透后再操作。

**Q: 按热键无法呼出输入栏？**
A: 请确认热键设置正确（在设置中点击热键标签后直接按组合键重新设置）。注意全局热键可能被其他程序或游戏拦截。

**Q: 热键按下后游戏聊天框也打开了？**
A: 翻译器会通过 AttachThreadInput + SetForegroundWindow 抢占焦点，输入焦点会从游戏转移到翻译器。如果仍有问题，尝试更换热键组合。

**Q: 倒计时后消息没有发送到游戏？**
A: 请确保在倒计时期间（屏幕显示"大模型翻译完成等待"时）点击了游戏窗口，让游戏获得输入焦点。同时确认游戏聊天键设置与游戏中一致。

**Q: 发送消息时提示"翻译失败"？**
A: 请检查 API 配置是否正确，确保 API 可以正常访问。
