# Logging Feature Design

> **日期:** 2026-06-14 | **状态:** Approved

## 目标

在不改变现有功能的前提下，增加日志系统：
- 日志写入文件持久保存
- 在设置对话框中提供日志查看标签页
- 记录关键事件：TMP 系统信息、LLM/Baidu 翻译连通性、报错、热键、更新

## 架构

```
logger.py (新增模块)
  ├─ 写文件: Documents/ETS2 Translator/logs/translator_YYYY-MM-DD.log
  ├─ 内存缓冲: 最近 500 条日志行 → 供 UI 读取
  ├─ 自动轮转: 保留最近 7 个日志文件，单文件 > 2MB 自动切分
  │
  ▼ 模块调用
main.py, monitor.py, translator.py, overlay.py, hotkey_manager.py, update.py
  │
  ▼ UI 读取
main.py → SettingsDialog 新增「日志」标签页
  └─ 只读 Text widget，显示最近 500 条日志
  └─ 「打开日志文件夹」按钮
```

## 日志内容清单

### 系统 (System) — `[SYS]`
- 应用启动（版本号、Python 版本、配置路径、Documents 路径）
- 应用退出
- 单实例互斥检测（已有实例在运行）
- 配置保存

### TMP 监控 (Monitor) — `[TMP]`
- 日志目录状态（存在/不存在）
- 聊天日志文件找到/切换（文件名、大小）
- 监控启动/停止
- 日志行解析失败（TMP 格式变化时）

### LLM 翻译 (LLM) — `[LLM]`
- 启动时 API 连通性测试结果（成功 → 模型名+延迟；失败 → 错误详情）
- 手动连通性测试结果（设置中点击测试按钮）
- API 调用失败（状态码 + 错误信息 + 重试情况）
- 每 50 条消息汇总统计（翻译数/缓存命中/跳过/节省率）

### 百度翻译 (Baidu) — `[BDU]`
- 启动时百度 API 连通性测试结果
- 手动连通性测试结果
- 百度 API 调用失败（错误码 + 错误信息）
- 混合模式下每 50 条汇总百度纠错数量

### 热键 (Hotkey) — `[HOT]`
- 全局热键注册（组合键）
- 热键变更

### 更新 (Update) — `[UPD]`
- 检查更新结果（发现新版/已是最新/网络失败）
- 下载进度/结果
- 更新安装结果

## 日志格式

```
YYYY-MM-DD HH:MM:SS [TAG] [LEVEL] message
```

示例：
```
2026-06-14 21:30:05 [SYS] [INFO] 翻译器启动 v1.2.0 | Python 3.12 | 配置: C:\Users\...\ETS2 Translator\config.json
2026-06-14 21:30:06 [LLM] [INFO] 连通 OK | gpt-4o-mini @ api.openai.com | 234ms
2026-06-14 21:30:06 [BDU] [INFO] 连通 OK | 百度翻译 API 标准版
2026-06-14 21:30:07 [TMP] [INFO] 聊天日志: chat_2026_06_14_log.txt (12.5 KB)
2026-06-14 21:31:22 [LLM] [WARN] 请求超时 (30s) | 已重试
2026-06-14 21:31:25 [LLM] [INFO] 翻译已恢复
2026-06-14 21:35:00 [UPD] [INFO] 检查更新: 已是最新 v1.2.0
```

## 日志文件管理

- 路径: `Documents/ETS2 Translator/logs/translator_YYYY-MM-DD.log`
- 轮转: 保留最近 7 个文件（按日期命名自动覆盖），单文件超过 2MB 切分为 `translator_YYYY-MM-DD_N.log`
- 启动时自动创建 `logs/` 目录

## 设置对话框 UI

设置对话框新增第 4 个标签页「📋 日志 / Logs」：

```
┌─ Settings / 设置 ────────────────────────────────────┐
│  [API 配置]  [快捷键]  [外观]  [📋 日志]               │
│                                                      │
│  ┌─ 日志内容 ───────────────────────────────────────┐ │
│  │ 2026-06-14 21:30:05 [SYS] INFO 翻译器启动 v1.2.0 │ │
│  │ 2026-06-14 21:30:06 [LLM] INFO 连通 OK           │ │
│  │ 2026-06-14 21:31:22 [LLM] WARN 请求超时          │ │
│  │ ...                                               │ │
│  └──────────────────────────────────────────────────┘ │
│                                                      │
│  [📂 打开日志文件夹]                                   │
└──────────────────────────────────────────────────────┘
```

- 日志 Text widget：只读、暗色背景、等宽字体、自动滚底
- 颜色区分：INFO 灰色 `#8b949e`、WARN 黄色 `#d29922`、ERROR 红色 `#f85149`
- 「打开日志文件夹」按钮 → `os.startfile(log_dir)` 打开资源管理器

## 模块变更

| 文件 | 动作 | 说明 |
|:---|:---|:---|
| `logger.py` | **新增** | 中央日志模块（文件写入 + 轮转 + 内存缓冲） |
| `main.py` | 修改 | App 初始化 logger；SettingsDialog 新增日志标签页 |
| `monitor.py` | 修改 | TMP 状态事件调用 logger |
| `translator.py` | 修改 | 连通性、错误、统计汇总调用 logger |
| `overlay.py` | 修改 | 热键事件调用 logger |
| `update.py` | 修改 | 更新事件调用 logger |

## 非功能约束

- 不改变任何现有 API、配置项、UI 布局（仅在设置对话框新增标签页）
- 日志写入不能阻塞主线程（使用 `logging` 模块的异步特性或独立线程）
- 日志文件不能无限制增长（最多保留 7 个文件 × 2MB = 14MB）
- 不记录任何聊天原文内容（仅记录翻译系统事件）
